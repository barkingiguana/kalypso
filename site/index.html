One trusted root CA. Short-lived certificates. A REST API your containers can call. Like Let's Encrypt, but for localhost.

Built for developers

Everything you need to run HTTPS locally, nothing you don't.

Short-Lived Certs

24-hour default, 7-day max. If a dev cert leaks, it expires before anyone can use it.

📦

Docker Compose Sidecar

Add one service to your compose file. Every container gets trusted HTTPS.

🔄

Auto-Renewal

The refresh sidecar renews certs at 50% lifetime. Certs never expire during dev sessions.

REST API

Simple JSON API. No client library needed — curl is enough.

🚀

SDKs

Up and running in 60 seconds

Trust once, issue forever.

Terminal 
# Install
pip install kalypso

# Create your root CA (once)
kalypso init

# Trust it (macOS)
sudo security add-trusted-cert -d -r trustRoot \
  -k /Library/Keychains/System.keychain \
  ~/.kalypso/ca-cert.pem

# Issue a cert — valid 24 hours
kalypso issue myapp.local

# Done. Use cert.pem and key.pem
docker-compose.yml 
services:
  kalypso:
    image: kalypso/kalypso:latest
    volumes:
      - kalypso-data:/data
      - certs:/certs
    ports:
      - "8200:8200"

  web:
    image: nginx:alpine
    volumes:
      - certs:/etc/nginx/certs:ro

volumes:
  kalypso-data:
  certs:

Your language, your way

Zero-dependency SDKs, or just use curl.

Python
from kalypso_sdk import KalypsoClient

client = KalypsoClient()
cert = client.issue("myapp.local")
cert.save("cert.pem", "key.pem")
Ruby
require "kalypso"

client = Kalypso::Client.new
cert = client.issue("myapp.local")
cert.save("cert.pem", "key.pem")
Go
client := kalypso.NewClient("http://localhost:8200")
cert, _ := client.Issue(ctx, "myapp.local")
cert.Save("cert.pem", "key.pem")
curl
curl -X POST http://localhost:8200/certificates \
  -H 'Content-Type: application/json' \
  -d '{"domains": ["myapp.local"]}'
