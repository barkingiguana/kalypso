Docker & Compose

Add two services to your docker-compose.yml. Certs are issued automatically, renewed automatically, and written to a shared volume. Your web server just reads them.

30-Second Setup

Copy this into your docker-compose.yml:

services:
  # 1. The CA server
  kalypso:
    image: kalypso/kalypso:latest
    ports:
      - "8200:8200"
    volumes:
      - kalypso-data:/data
    healthcheck:
      test: ["CMD", "python", "-c", "import urllib.request; urllib.request.urlopen('http://localhost:8200/health')"]
      interval: 5s
      timeout: 3s
      retries: 5

  # 2. The magic sidecar — issues certs and renews them automatically
  certs:
    image: kalypso/kalypso:latest
    command: ["sidecar"]
    depends_on:
      kalypso:
        condition: service_healthy
    environment:
      KALYPSO_DOMAINS: "myapp.local,*.myapp.local"
    volumes:
      - certs:/certs

  # 3. Your web server — just mount the certs volume
  web:
    image: nginx:alpine
    depends_on:
      certs:
        condition: service_started
    ports:
      - "443:443"
      - "80:80"
    volumes:
      - certs:/etc/nginx/certs:ro
      - ./nginx.conf:/etc/nginx/conf.d/default.conf:ro

volumes:
  kalypso-data:
  certs:

That's it. Run docker compose up -d and you have HTTPS.

How It Works

  1. kalypso starts and creates a root CA (or loads an existing one from the kalypso-data volume)
  2. certs sidecar waits for kalypso to be healthy, then requests a cert for your domains
  3. Sidecar writes cert.pem, key.pem, ca.pem, and fullchain.pem to the shared certs volume
  4. web (nginx) reads certs from the shared volume
  5. Sidecar sleeps until 50% of cert lifetime, then re-issues automatically

nginx.conf

Create an nginx.conf next to your docker-compose.yml:

server {
    listen 80;
    server_name myapp.local;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name myapp.local;

    ssl_certificate     /etc/nginx/certs/cert.pem;
    ssl_certificate_key /etc/nginx/certs/key.pem;

    ssl_protocols TLSv1.2 TLSv1.3;

    location / {
        # proxy_pass, root, etc. — your app here
        return 200 'Hello from HTTPS!';
        add_header Content-Type text/html;
    }
}

Trust the CA on Your Host

Your browser needs to trust the Kalypso CA. This is a one-time step:

# Download the CA certificate (it's raw PEM, no jq needed)
curl http://localhost:8200/ca.pem > kalypso-ca.pem

# macOS
sudo security add-trusted-cert -d -r trustRoot \
  -k /Library/Keychains/System.keychain kalypso-ca.pem

# Ubuntu / Debian
sudo cp kalypso-ca.pem /usr/local/share/ca-certificates/kalypso.crt
sudo update-ca-certificates

# Fedora / Arch
sudo trust anchor kalypso-ca.pem

# Or if you have kalypso installed locally:
pip install kalypso
kalypso init          # skip if using Docker CA
sudo kalypso trust    # auto-detects your OS

Then add to /etc/hosts:

127.0.0.1 myapp.local

Visit https://myapp.local — green lock, zero warnings.

Sidecar Configuration

The sidecar is configured entirely through environment variables:

VariableDefaultDescription
KALYPSO_DOMAINS—Comma-separated domains (required)
KALYPSO_SERVERhttp://kalypso:8200Kalypso server URL
KALYPSO_CERT_DIR/certsWhere to write cert files
KALYPSO_HOURS24Cert lifetime in hours
KALYPSO_RELOAD_CMD—Command to run after renewal

Output files

The sidecar writes these files to KALYPSO_CERT_DIR:

FileContents
cert.pemThe leaf certificate
key.pemThe private key
ca.pemThe CA certificate
fullchain.pemcert.pem + ca.pem combined

Inter-Container TLS

If container A needs to call container B over HTTPS with a Kalypso cert, tell your language's HTTP client to trust the CA:

services:
  my-service:
    environment:
      - REQUESTS_CA_BUNDLE=/certs/ca.pem           # Python requests
      - NODE_EXTRA_CA_CERTS=/certs/ca.pem          # Node.js
      - SSL_CERT_FILE=/certs/ca.pem                # Ruby, Go, curl
    volumes:
      - certs:/certs:ro
