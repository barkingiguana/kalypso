Docker & Compose

Add a kalypso.domains label to any service. Kalypso auto-discovers it, issues certs, and reloads your server. That's it.

Magic Mode (Recommended)

Mount the Docker socket and a shared certs volume. Kalypso does the rest.

services:
  kalypso:
    image: kalypso/kalypso:latest
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
      - kalypso-data:/data
      - certs:/certs
    ports:
      - "8200:8200"

  web:
    image: nginx:alpine
    labels:
      kalypso.domains: "myapp.local,*.myapp.local"
    volumes:
      - certs:/etc/nginx/certs:ro
      - ./nginx.conf:/etc/nginx/conf.d/default.conf:ro
    ports:
      - "443:443"

volumes:
  kalypso-data:
  certs:

Run docker compose up -d. Done.

What happens automatically:

  1. Kalypso starts and creates a root CA
  2. Kalypso detects the Docker socket and watches for containers with kalypso.* labels
  3. It discovers your web container, issues a cert for myapp.local
  4. Writes cert.pem, key.pem, ca.pem, fullchain.pem to the shared certs volume
  5. Detects nginx and runs nginx -s reload automatically
  6. Renews at 50% lifetime (every 12h by default)

nginx.conf

server {
    listen 443 ssl;
    server_name myapp.local;
    ssl_certificate     /etc/nginx/certs/cert.pem;
    ssl_certificate_key /etc/nginx/certs/key.pem;
    location / {
        # your app here
    }
}

Labels Reference

Add these labels to any service you want Kalypso to manage:

LabelRequiredDefaultDescription
kalypso.domainsYes—Comma-separated domains
kalypso.hoursNo24Cert lifetime in hours
kalypso.cert_dirNoautoSubdirectory under /certs
kalypso.reloadNoauto-detectedReload command to exec

Auto-Reload Detection

Kalypso auto-detects the reload command from the container image:

ImageAuto-detected reload
nginx:*nginx -s reload
httpd:* / apache:*apachectl graceful
haproxy:*kill -USR2 1
caddy:*caddy reload
traefik:*None needed (watches files)

For other images, add kalypso.reload label:

labels:
  kalypso.domains: "myapp.local"
  kalypso.reload: "kill -HUP 1"

Trust the CA on Your Host

When Kalypso starts, it prints trust instructions in the Docker logs. Or just run:

# Download the CA cert (raw PEM, no jq needed)
curl http://localhost:8200/ca.pem > /tmp/kalypso-ca.pem

# macOS
sudo security add-trusted-cert -d -r trustRoot \
  -k /Library/Keychains/System.keychain /tmp/kalypso-ca.pem

# Ubuntu / Debian
sudo cp /tmp/kalypso-ca.pem /usr/local/share/ca-certificates/kalypso.crt
sudo update-ca-certificates

# Fedora / Arch
sudo trust anchor /tmp/kalypso-ca.pem

Then add to /etc/hosts:

127.0.0.1 myapp.local

Sidecar Mode (Alternative)

If you don't want to mount the Docker socket, use the sidecar pattern instead. Add a second Kalypso container with command: ["sidecar"]:

services:
  kalypso:
    image: kalypso/kalypso:latest
    volumes: [kalypso-data:/data]
    ports: ["8200:8200"]

  certs:
    image: kalypso/kalypso:latest
    command: ["sidecar"]
    environment:
      KALYPSO_DOMAINS: "myapp.local"
    volumes: [certs:/certs]

  web:
    image: nginx:alpine
    volumes:
      - certs:/etc/nginx/certs:ro

Sidecar Environment Variables

VariableDefaultDescription
KALYPSO_DOMAINS—Comma-separated domains (required)
KALYPSO_SERVERhttp://kalypso:8200Kalypso server URL
KALYPSO_CERT_DIR/certsOutput directory
KALYPSO_HOURS24Cert lifetime in hours
KALYPSO_RELOAD_CMD—Post-renewal command

Inter-Container TLS

If container A needs to call container B over HTTPS:

services:
  my-service:
    environment:
      - REQUESTS_CA_BUNDLE=/certs/ca.pem           # Python requests
      - NODE_EXTRA_CA_CERTS=/certs/ca.pem          # Node.js
      - SSL_CERT_FILE=/certs/ca.pem                # Ruby, Go, curl
    volumes:
      - certs:/certs:ro
