Docker & Compose

Add a kalypso.domains label to any container. Kalypso auto-discovers it, issues certs, injects them directly, trusts the CA inside the container, and reloads. No shared volumes. No shared networks. Just a label.

Zero-Config Mode (Recommended)

Run Kalypso once on your machine. Every project gets HTTPS automatically — completely separate Compose stacks, no coupling.

1. Start Kalypso (once per machine)

# kalypso/docker-compose.yml — run this once
services:
  kalypso:
    image: kalypso/kalypso:latest
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
      - kalypso-data:/data
    ports:
      - "8200:8200"

volumes:
  kalypso-data:

2. Add one label to any project

# webapp/docker-compose.yml — a totally separate project
services:
  web:
    image: nginx:alpine
    labels:
      kalypso.domains: "myapp.local,*.myapp.local"
    ports:
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf:ro

That's it. No shared volumes, no shared networks, no depends_on. Run docker compose up -d in each directory.

What happens automatically:

  1. Kalypso detects your web container via the Docker socket
  2. Issues a cert for myapp.local
  3. Knows it's nginx — injects cert.pem and key.pem into /etc/nginx/ssl/ directly via the Docker API
  4. Injects the CA into the container's system trust store
  5. Runs nginx -s reload automatically
  6. Renews at 50% lifetime (every 12h by default)

nginx.conf

server {
    listen 443 ssl;
    server_name myapp.local;
    # Kalypso knows these paths for nginx — no config needed
    ssl_certificate     /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;
    location / {
        # your app here
    }
}

Labels Reference

Add these labels to any container you want Kalypso to manage:

LabelRequiredDefaultDescription
kalypso.domainsYes—Comma-separated domains
kalypso.cert-pathNoauto-detectedOverride cert path inside container
kalypso.key-pathNoauto-detectedOverride key path inside container
kalypso.ca-pathNoauto-detectedOverride CA cert path inside container
kalypso.reloadNoauto-detectedReload command to exec
kalypso.hoursNo24Cert lifetime in hours
kalypso.trustNotrueSet to "false" to skip CA trust store injection
kalypso.autoNotrueSet to "false" to disable image-based auto-detection

Image Profiles

Kalypso auto-detects cert paths and reload commands from the container image:

ImageCert pathKey pathReload command
nginx:*/etc/nginx/ssl/cert.pem/etc/nginx/ssl/key.pemnginx -s reload
httpd:* / apache:*/usr/local/apache2/conf/server.crt...server.keyapachectl graceful
haproxy:*/usr/local/etc/haproxy/certs/cert.pem...key.pemkill -USR2 1
caddy:*/etc/caddy/certs/cert.pem...key.pemNone (file watcher)
traefik:*/etc/traefik/certs/cert.pem...key.pemNone (file watcher)
Everything else/etc/ssl/kalypso/cert.pem...key.pemNone

For unknown images, use labels to tell Kalypso where certs go:

labels:
  kalypso.domains: "myapp.local"
  kalypso.cert-path: "/app/certs/cert.pem"
  kalypso.key-path: "/app/certs/key.pem"
  kalypso.reload: "kill -HUP 1"

Server Flags

Control what Kalypso does automatically with flags on the serve command (or env vars):

FlagEnv varDefaultDescription
--auto-inject / --no-auto-injectKALYPSO_AUTO_INJECTonInject certs directly into containers
--auto-trust / --no-auto-trustKALYPSO_AUTO_TRUSTonInject CA into container trust stores
--auto-reload / --no-auto-reloadKALYPSO_AUTO_RELOADonAuto-detect and run reload commands

Turn off --no-auto-inject to fall back to the shared-volume approach described below.

Trust the CA on Your Host

Kalypso auto-trusts the CA inside containers. For your host machine (browser, curl, etc.):

# Download the CA cert
curl http://localhost:8200/ca.pem > /tmp/kalypso-ca.pem

# macOS
sudo security add-trusted-cert -d -r trustRoot \
  -k /Library/Keychains/System.keychain /tmp/kalypso-ca.pem

# Ubuntu / Debian
sudo cp /tmp/kalypso-ca.pem /usr/local/share/ca-certificates/kalypso.crt
sudo update-ca-certificates

# Fedora / Arch
sudo trust anchor /tmp/kalypso-ca.pem

Then add to /etc/hosts:

127.0.0.1 myapp.local

Shared Volume Mode

If you prefer shared volumes over direct injection (or need read-only containers), use --no-auto-inject and mount a shared certs volume:

services:
  kalypso:
    image: kalypso/kalypso:latest
    command: ["--data-dir", "/data", "serve", "--no-auto-inject"]
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
      - kalypso-data:/data
      - certs:/certs
    ports:
      - "8200:8200"

  web:
    image: nginx:alpine
    labels:
      kalypso.domains: "myapp.local,*.myapp.local"
    volumes:
      - certs:/etc/nginx/certs:ro
      - ./nginx.conf:/etc/nginx/conf.d/default.conf:ro
    ports:
      - "443:443"

volumes:
  kalypso-data:
  certs:

Sidecar Mode

If you don't want to mount the Docker socket at all, use the sidecar pattern:

services:
  kalypso:
    image: kalypso/kalypso:latest
    volumes: [kalypso-data:/data]
    ports: ["8200:8200"]

  certs:
    image: kalypso/kalypso:latest
    command: ["sidecar"]
    environment:
      KALYPSO_DOMAINS: "myapp.local"
    volumes: [certs:/certs]

  web:
    image: nginx:alpine
    volumes:
      - certs:/etc/nginx/certs:ro

Inter-Container TLS

Kalypso auto-trusts the CA inside containers. But if you need a specific app runtime to trust it (e.g. Python requests, Node.js), set the env var:

services:
  my-service:
    environment:
      - REQUESTS_CA_BUNDLE=/etc/ssl/kalypso/ca.pem    # Python requests
      - NODE_EXTRA_CA_CERTS=/etc/ssl/kalypso/ca.pem   # Node.js
      - SSL_CERT_FILE=/etc/ssl/kalypso/ca.pem         # Ruby, Go, curl
